## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| wire → models.py enum columns | Python enum values must match Postgres ENUM literals exactly; mismatch is a data corruption vector |
| Alembic DDL → Postgres schema | Hand-written ENUM DDL creates the types the ORM columns reference; migration failures corrupt schema state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-01 | Tampering | models.py ENUM column `create_type=True` (default) | mitigate | Use `PG_ENUM(..., create_type=False)` on all enum columns; test `Base.metadata.create_all` in pg_engine integration fixture to confirm no "type already exists" error (RESEARCH Pitfall 6) |
| T-3-02 | Information Disclosure | CredentialNotification fields in port file | mitigate | Port file defines the shape but never instantiates secrets; field name `admin_password` is a struct field only — add a `# sensitive: never log, never serialize to DB` comment in the dataclass docstring |
| T-3-03 | Tampering | Alembic ENUM schema-qualification omission | mitigate | Every `op.execute("CREATE TYPE ...")` uses schema-qualified name `provisioning.instance_status`; all `op.create_table(...)` calls include `schema="provisioning"` — verify with `make psql` after migration |
| T-3-SC | Tampering | npm/pip/cargo installs | accept | No new packages this plan — all dependencies already in uv.lock; no install tasks |
</threat_model>
