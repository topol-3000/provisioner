<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| provisioning_worker → provisioning.event_outbox (JSONB payload) | Envelope payload is written as JSONB by OutboxRepo.enqueue; rebuilt by relay at publish time. Trust boundary: what enters the JSONB column is controlled by this process only (no external writer). |
| provisioning.event_outbox → events.instance (XADD) | The relay rebuilds the envelope from JSONB and publishes to the Valkey stream. platform-api reads events.instance — it is a consumer, not an attacker. But the relay must not leak secrets into the stream. |
| InstanceProvisionedPayload fields | Only non-credential instance metadata. Credentials (admin_password) flow only through NotificationTransport, never into events or logs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | InstanceProvisionedPayload | mitigate | model has no admin_password or credential fields (D-09); frozen+extra="forbid" rejects any accidental field addition at validation time; docstring explicitly states "no credentials" |
| T-04-02 | Information Disclosure | outbox_relay.py last_error logging | mitigate | log.warning uses str(exc) not repr(exc) for the log line; repr(exc) is used only for the last_error DB column (bounded by _truncate); RedisError str() contains no secrets |
| T-04-03 | Tampering | UNIQUE(envelope_id) backstop | accept | Primary mechanism is is_first_ready guard (D-01); UNIQUE(envelope_id) + ON CONFLICT DO NOTHING is defense-in-depth. Fresh ULID per emit means collision is astronomically unlikely; consumer-side dedupe on envelope.id is the final layer |
| T-04-04 | Denial of Service | relay poll loop | mitigate | "relay must never die" pattern wraps each iteration in try/except + log.exception; SKIP LOCKED prevents one stuck row from blocking the batch |
| T-04-05 | Elevation of Privilege | valkey_url injection | accept | valkey_url is RedisDsn-typed pydantic field; validated at startup; internal network only; no user-controlled input |
| T-04-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed in this phase (RESEARCH.md §Package Legitimacy Audit confirms zero new deps). slopcheck N/A. |
</threat_model>
