# Phase 2 Plan 2: ProcessedEvent ORM + processed_event Migration Summary

The DB persistence layer for idempotency dedupe: a `ProcessedEvent` SQLAlchemy 2.0
mapped class on `provisioning.processed_event` (composite PK `(event_id,
consumer_group)`, `processed_at TIMESTAMPTZ` defaulting to `NOW()`), the first
hand-reviewed migration in the single provisioning Alembic tree, `env.py` wired to
`Base.metadata` for future autogenerate, and a `consumer_reclaim_min_idle_ms`
Settings tunable for the XAUTOCLAIM reclaim window.

## What Was Built

**Task 1 (TDD) — ProcessedEvent ORM + settings**
- `models.py`: replaced the Phase-1 docstring stub with `Base(DeclarativeBase)`,
  `_SCHEMA = "provisioning"`, and `ProcessedEvent` (`__tablename__ =
  "processed_event"`, `__table_args__` ClassVar dict with the schema). Columns:
  `event_id: Mapped[str]` `String(26)` PK, `consumer_group: Mapped[str]` `Text` PK,
  `processed_at: Mapped[datetime]` `TIMESTAMP(timezone=True)` with
  `server_default=func.now()`. `__all__ = ["Base", "ProcessedEvent"]`.
- `settings.py`: added `consumer_reclaim_min_idle_ms: int` (default `60_000`,
  `ge=1_000`) after `consumer_name`.
- `.env.example`: added `CONSUMER_RECLAIM_MIN_IDLE_MS=60000`.
- Tests: `tests/provisioning/test_models.py` (tablename, schema, column set,
  composite PK, metadata registration) + two `tests/test_settings.py` cases
  (default 60_000, rejects 500).

**Task 2 — Alembic migration + env.py**
- `env.py`: `target_metadata = None` → `Base.metadata` (imports `Base` from
  `modules/provisioning/models`).
- Scaffolded via `make revision name="add processed_event"`; autogenerate
  correctly detected the schema-qualified table. Replaced the autogenerated
  bodies with clean hand-authored `upgrade()`/`downgrade()`: `op.create_table`
  with `sa.PrimaryKeyConstraint("event_id", "consumer_group")` and
  `schema="provisioning"`; `op.drop_table(..., schema="provisioning")`.
  `down_revision = None` (first migration, rev `0e3f3be0f9ad`).
- `make migrate` ran against live Postgres; the table, columns, and composite PK
  were verified in `information_schema` / `pg_index`.
