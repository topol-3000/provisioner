## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| handler → session (dedupe txn) | Handler stages rows; commit happens outside handler body. Post-commit enqueue must wait for commit to avoid phantom jobs (Pitfall 1). |
| tasks.py → structlog | Any *_password, *_token, *_secret kwarg is a credential leak. NEVER bind secrets to logger. |
| ready_at guard → NotificationTransport | The guard must be atomic with the ready_at update — read and set in the same DB transaction to prevent double delivery (Pitfall 4, D-13). |
| create_instance_task → consumer loop | Unhandled exception in task must not propagate to the broker's listen() loop, which would crash the consumer concern. |
